Linux Command Line Basics



Introduction

Running bioinformatics analyses entirely on graphical user interfaces is very easy and convenient, particularly if you are new to bioinformatics or are a scientist who just needs to run a few analyses but not dive into the fundamentals. In fact, this realization has resulted in the development of several useful and freely available GUI-based bioinformatics tools, in part to expand scientists' access to these analyses. However, more often than not, these GUI tools are insufficient for the needs of day to day bioinformatics. Enter Linux and the command line.

For a number of reasons that are outside the scope of this short course, the combination of the Linux operating system and the command line interface unlocks powerful bioinformatics capabilities that are not possible on GUI-based tools and programs. We have designed this course to introduce you to the fundamentals of the Linux command line interface and provide you with the needed information for you to get started using bioinformatics tools on the command line interface.



What is the command line?

The command line interface or shell or terminal is basically a program, like most things on a computer. The command line provides a text-based user interface that, in its most basic use case, allows a user to type and run 'commands' (or other programs) that are executed to perform definite tasks. This might seem like a lot to wrap your head around, but it really is simple. Think of your computer, which probably has a GUI. On your computer you have several installed programs that can be "launched" or "executed" when you click on them. These GUI programs often depend on these "click" actions in order to execute what the user intends. You click to play videos, click to pause, click to open files and click to close them. In there exists the major difference between CLI and GUI: in the command line interface, you need to type and run commands to execute them. The executed programs run are either built-in commands or external installed programs. We would see some examples as we progress through the course.

Linux and MacOS systems have a default or built-in unix shell or terminal. On Windows machines, however, the default command line interface is not unix-based and to run Linux commands on these machines, you would need to install a Linux shell. Some available options are Windows subsystem for Linux. Okay we have laid the foundations. Now let us launch our shells and dive right in!



The Linux file structure

The Linux file system is a wide topic. We could introduce file structure, Linux as a multiuser operating system, user files, system files, file permissions etc and we still would not have scratched the surface. The good news is we do not need to get into those intricacies in order to get started. Here would only look at the Linux file structure as that is crucial to understanding the subsequent topics. First, let's define files and directories. Files are simply objects that hold bits of information; this could be data, settings or programs. Directories are simply files (you would probably know them as "folders") that contain other files or directories. The Linux file hierarchy is the structure or order of directories and files within the Linux system. The conserved hierarchical structure of this arrangement within Linux systems is very important for many reasons but one that requires mention here is the fact that it allows programs to be able to locate system configuration files across different Linux distributions. As you use and get more familiar with Linux systems, you would understand why this is very important. For now it would suffice to note that this structure is critical to the optimal functioning of the Linux system. Most Linux systems would contain a directory structure as shown below.

Let us explain some core concepts regarding this structure before exploring the important aspects seen in the figure above. Directories that contain other directories or files are referred to as parent directories relative to the objects they contain while the directories they contain are referred to as child directories relative to the parent. The relationship between parent and children directories are denoted by forward slashes which separate a parent from a direct child. The top-most directory in all systems is the root directory, denoted by a forward slash (/). All other directories in the system are contained within this directory.
